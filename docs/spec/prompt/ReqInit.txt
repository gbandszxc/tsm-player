你是一个资深的 Android 开发者，专精于 Android TV / 大屏设备应用开发。现在请为我设计并输出一个极简的 Android TV 音乐播放器 App 的完整需求规格 + 核心实现思路 + 关键代码框架提示。

App 目标用户：使用安卓电视（无 Google Play，只能侧载 APK）的用户，音乐文件全部整理好放在 NAS 上，通过 SMB 共享访问。

核心需求（必须 100% 满足，不能妥协）：
1. **界面与交互**：
   - 专为 Android TV 优化：使用 Leanback 库（RowsSupportFragment / BrowseSupportFragment 等）或 Jetpack Compose for TV，确保大屏友好、遥控器（D-Pad）导航顺畅、无触摸依赖。
   - 极简风格：首页直接是 SMB 服务器浏览器（文件树视图），不需要任何音乐库扫描、自动索引、艺术家/专辑分类等功能。
   - 支持手动浏览文件夹：像文件管理器一样，列出文件夹和音乐文件（MP3/FLAC 等常见格式），支持进入子文件夹、返回上级。

2. **SMB 网络支持**：
   - 支持连接 SMB/CIFS 服务器（v2/v3 协议，v1 可选兼容）。
   - 用户手动输入：服务器 IP（如 192.168.31.233）、共享名（如 Banana）、子路径（如 h/DLsite）、用户名/密码（支持 Guest/匿名）。
   - 连接后显示文件列表（文件夹 + 音频文件），支持递归浏览。
   - 使用可靠库：推荐 jcifs-ng 或 libsmbclient-android（避免旧 jcifs 问题）。
   - 连接失败时友好提示（网络问题、认证失败、路径无效等）。

3. **音乐播放**：
   - 点击音频文件直接播放单曲；长按文件夹或多选可播放整个文件夹（顺序或随机，生成临时播放列表）。
   - 使用 ExoPlayer 或 Media3 作为播放核心，支持 gapless 无缝播放、高分辨率音频（FLAC 等）。
   - 播放界面：大屏显示当前歌曲名、艺术家、专辑、进度条、播放/暂停/下一首/上一首（遥控器适配）。
   - 支持后台播放 + 通知栏/锁屏控制。

4. **封面显示**：
   - 优先读取内嵌专辑封面（ID3v2 APIC 或 FLAC PICTURE）。
   - 次选：同文件夹下的 folder.jpg / cover.jpg / front.jpg 等常见文件名。
   - 播放界面全屏或大图显示封面（如果没有则显示默认图标）。

5. **歌词显示**（最关键痛点）：
   - 支持**外挂 LRC**：同文件夹、同文件名（song.mp3 → song.lrc），UTF-8 编码，带时间戳的同步歌词。
   - 支持**内嵌歌词**：从音频标签读取 UNSYNCED LYRICS 或 LYRICS 字段（ID3v2 USLT / PRIV 等）。
   - 播放时在界面下方或全屏模式显示滚动歌词（karaoke 风格，高亮当前行）。
   - 必须兼容亚洲语言（中文/日文/韩文）字符，不乱码、不消失（建议嵌入 Noto Sans CJK 字体或强制使用系统支持 Unicode 的字体）。
   - 无需在线搜索/下载歌词，只读本地文件/标签。

6. **其他必须/强需求**：
   - 无需任何互联网权限（除了 SMB 网络访问），完全离线可用。
   - 无广告、无崩溃、无复杂设置页面。
   - 支持常见音频格式：MP3, FLAC, AAC, WAV 等。
   - 最小化依赖：目标 APK 体积小，便于侧载。
   - Android TV 最低 API 21+（兼容老电视盒子）。

7. **不包含的功能**（明确排除，避免多余）：
   - 自动扫描音乐库、MediaStore 集成、标签编辑、在线歌词搜索、云盘集成、EQ/音效、播放列表管理（除临时文件夹播放外）、小部件等。
