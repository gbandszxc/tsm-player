# 下一步计划（无编码版）

## 目标
把当前“Leanback 浏览骨架 + 假数据”升级为可连接真实 NAS、可播放、可显示封面和歌词的可侧载 Android TV 应用。

## Phase 1: 真实 SMB 接入

1. 引入 `jcifs-ng` 依赖并替换 `FakeSmbRepository`。
2. 实现连接配置持久化（DataStore）。
3. 实现目录递归浏览与错误分级提示：
   - 认证失败
   - 服务器不可达
   - 共享名不存在
   - 路径无效
4. 明确协议策略：默认 SMB2/3，预留 SMB1 兼容开关（可选）。

交付物：
- 可输入 NAS 参数并真实列出目录和音频文件。

## Phase 2: 播放核心

1. 接入 Media3 ExoPlayer。
2. 文件点击播放（单曲）与“整目录播放（顺序/随机）”。
3. 构建临时播放列表，不做长期播放列表管理。
4. 建立后台播放能力（MediaSession + Notification）。

交付物：
- Android TV 上可稳定播放 SMB 音频流。

## Phase 3: 元数据与封面

1. 元数据读取：标题/艺术家/专辑。
2. 封面优先级：
   - 内嵌封面（APIC/FLAC PICTURE）
   - 同目录 `folder.jpg`/`cover.jpg`/`front.jpg`
   - 默认占位图
3. 优化加载策略（线程与缓存）避免主线程卡顿。

交付物：
- 播放页显示正确曲目信息与封面。

## Phase 4: 歌词能力

1. 外挂 LRC：同名同目录匹配。
2. 内嵌歌词：读取 USLT/LYRICS 字段。
3. 歌词时间轴同步：
   - 解析 offset
   - 二分查找当前行
4. CJK 文本显示验证（中/日/韩）与字体策略评估。

交付物：
- 播放时可滚动并高亮同步歌词。

## Phase 5: TV 交互与可用性收敛

1. 遥控器焦点流全面检查（进入/返回/播放控制）。
2. 大屏布局细化（列表密度、播放页信息层级、错误提示可见性）。
3. 弱网与异常恢复策略（重试、超时、断流恢复）。

交付物：
- TV 交互稳定，异常场景可自恢复或明确提示。

## Phase 6: 打包与交付

1. Debug/Release 构建流水固定。
2. 配置 release 签名（keystore）并产出可安装 release 包。
3. 形成最终测试清单：
   - SMB 连接
   - 播放稳定性
   - 封面回退链路
   - 歌词同步精度
   - D-Pad 导航

交付物：
- 可侧载、可验收的 release APK 与交付文档。

## 风险与优先级

1. 最高优先：SMB 真实连接稳定性、播放器稳定性。
2. 次优先：歌词同步精度、CJK 显示兼容。
3. 最后优化：视觉样式与动效细节。
