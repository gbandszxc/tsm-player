# 文件列表页元信息与排序功能设计

## 背景

TSM Player 当前浏览页的文件列表只显示类型图标和名称，缺少文件大小、最后修改时间，也没有明确的排序切换能力。对 TV 遥控器场景来说，用户需要在不进入详情页的前提下快速判断文件信息，并且能在名称、大小、时间之间切换正序和倒序。

当前顶部工具区同时存在“刷新当前目录”和“重试连接”两个按钮，但用户真正需要的是一个统一的刷新入口，以及一个清晰的排序选择控件。标题栏本身不应成为可聚焦元素，以免干扰遥控器操作。

## 目标

- 文件列表页每行增加显示名称、文件大小、最后修改时间。
- 顶部工具区只保留“刷新当前目录”和“排序”两个可聚焦控件。
- 排序默认使用“文件名升序”。
- 排序支持参照列和正倒序组合，共 6 个选项。
- 目录始终排在文件前，`..` 返回上一级始终固定在最上方。
- 排序切换后保持焦点尽量回到原先那一项，避免 TV 遥控器体验跳变。

## 非目标

- 不在本轮增加更多筛选能力，如“只看目录”“只看音频类型”。
- 不在本轮持久化排序偏好到设置或本地存储。
- 不新增全屏排序设置页。
- 不改变当前 SMB 浏览的基本进入目录、返回上级、快速定位和播放流程。

## 交互设计

### 顶部工具区

文件浏览区域顶部工具区调整为仅两项：

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 文件浏览                                                                     │
│ 当前路径：/music/YOASOBI                                                     │
│ [刷新当前目录]  [文件名 ↑ ▾]                                                 │
└──────────────────────────────────────────────────────────────────────────────┘
```

- “文件浏览”标题保持纯文本，不可聚焦。
- 当前路径文本保持不可聚焦。
- “刷新当前目录”承担普通刷新和加载失败后的重试职责，不再单独保留“重试连接”按钮。
- “排序”默认文案显示当前排序方式，例如 `文件名 ↑`、`修改时间 ↓`。

### 排序选择控件

排序控件视觉上表现为下拉框，但交互上采用轻量锚定式选择面板，而不是全屏 modal。

```text
┌────────────────────────────┐
│ [文件名 ↑ ▾]               │
│  ┌──────────────────────┐  │
│  │ 文件名 ↑            │  │
│  │ 文件名 ↓            │  │
│  │ 文件大小 ↑          │  │
│  │ 文件大小 ↓          │  │
│  │ 修改时间 ↑          │  │
│  │ 修改时间 ↓          │  │
│  └──────────────────────┘  │
└────────────────────────────┘
```

- 按 OK 打开排序选择面板。
- 当前已选项只用选中样式区分，不追加“当前”等辅助文字。
- 用户选中任一选项后，面板立即自动收起并应用排序。
- Back 在面板展开时直接关闭面板并把焦点还给排序按钮。
- 如果现有 `TsmModalCoordinator` 不适合轻量锚定面板，需要新增浏览页专用的轻量选择面板实现，但视觉 token 仍需复用 `DESIGN.md` 中的公共规范。

### 列表行

文件列表行改为四段式信息布局：

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ > [DIR] ..                      --                --                         │
│   [DIR] Album A                 --                2026-05-30 21:15:08       │
│   [AUD] 01 - intro.flac         25.4 MB           2026-05-12 13:20:45       │
│   [AUD] 02 - theme.mp3          8.7 MB            2026-05-01 18:03:11       │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 类型标识继续显示目录和音频的区分标记。
- 名称列占用最大剩余宽度。
- 大小列固定宽度。
- 时间列固定宽度。
- 所有列都保持单行显示，不换行。
- 名称列在未聚焦时使用单行省略；当前行获得焦点时，名称文本开启横向滚动以显示完整文件名。

## 数据设计

### `SmbEntry` 扩展

为避免大小和时间只能在 UI 层临时拼接，本次直接扩展领域模型：

- `sizeBytes: Long?`
- `lastModifiedAt: Long?`

语义约束：

- 文件：尽量同时填充 `sizeBytes` 和 `lastModifiedAt`。
- 目录：`sizeBytes = null`，`lastModifiedAt` 尽量填充目录最后修改时间。
- SMB 无法提供时间或大小时使用 `null`，由 UI 统一显示 `--`。

### 仓库层

`JcifsSmbRepository` 在枚举 `SmbFile` 时直接采集 SMB 元数据并写入 `SmbEntry`：

- `file.length()` -> `sizeBytes`
- `file.lastModified()` -> `lastModifiedAt`

兼容策略：

- 目录继续只返回可浏览目录和可播放音频。
- 对单个条目的元数据读取异常应以保守方式处理，优先降级为 `null`，避免整个目录因单个文件时间/大小读取失败而加载失败。
- `FakeSmbRepository` 同步补齐演示用大小和时间字段，保证调试和测试代码可编译、可运行。

## 排序设计

### 排序选项

ViewModel 中维护排序状态，枚举值固定为：

- `NAME_ASC`
- `NAME_DESC`
- `SIZE_ASC`
- `SIZE_DESC`
- `MODIFIED_ASC`
- `MODIFIED_DESC`

默认值为 `NAME_ASC`。

### 排序规则

最终展示顺序统一按以下规则生成：

1. `..` 返回上一级不参与排序，永远固定在第 1 项。
2. 目录组永远排在文件组前。
3. 目录组、文件组内部按当前排序选项排序。
4. 当主排序值相同时，回退到名称升序，保证顺序稳定。

各列比较规则：

- 文件名：按 `name.lowercase()` 比较。
- 文件大小：仅在文件组内部按 `sizeBytes` 比较；未知值排在该组末尾。
- 修改时间：目录组和文件组内部都按 `lastModifiedAt` 比较；未知值排在该组末尾。

未知值“固定排末尾”的含义是：

- 无论升序还是倒序，`null` 都不应因为翻转比较而跑到最前面。
- 用户会把 `--` 理解为“未知”，而不是“最小”或“最大”。

### 排序职责归属

排序统一放在 `TvBrowserViewModel`，而不是 Fragment 临时重排。

原因：

- 焦点锚点恢复要依赖最终展示顺序。
- 快速定位模式依赖最终列表索引。
- 列表重建指纹也应对准最终显示内容。
- 这样能避免“ViewModel 认为第 N 项是 A，但 UI 已重新排序成 B”的分层错位。

## 焦点与遥控器规则

- 顶部工具区焦点顺序固定为：`刷新当前目录 -> 排序`。
- 从工具区按下进入列表首项。
- 从列表首项按上返回“刷新当前目录”。
- 排序面板展开后，初始焦点落在当前已选项。
- 面板收起后，焦点回到排序按钮。
- 当排序切换时：
  - 若当前焦点对应的 `entry.fullPath` 仍存在于新列表，焦点恢复到该项的新位置。
  - 若当前项已不存在，则回退到第一个可见列表项；根目录为首项，子目录优先为 `..`。
- 快速定位模式始终基于排序后的列表工作，不为不同排序方式增加额外分支。

## 格式化规则

### 文件大小

- 文件大小显示单位按 `B / KB / MB / GB` 自动换算。
- 默认保留 1 位小数，若小数位为 0 则去掉尾随 `.0`。
- 目录显示 `--`。
- 未知大小显示 `--`。

### 修改时间

- 时间格式固定为 `yyyy-MM-dd HH:mm:ss`。
- 未知时间显示 `--`。
- 使用设备本地时区显示。

## UI 与样式要求

- 继续复用 `DESIGN.md` 中的深色面板、蓝色焦点和 `@color/ui_*`、`@dimen/ui_*`、`@style/Tsm*` token。
- 顶部工具区新增排序控件时，不新增散落的通用硬编码颜色、圆角和字号。
- 列表行新增大小列、时间列时，应为两列提供稳定宽度，避免排序切换时整行抖动。
- 名称列跑马灯仅在聚焦态启用，未聚焦时保持静止省略，避免长列表页面持续晃动。
- 如需新增轻量下拉面板背景或选中态 drawable，应优先复用现有 panel / list / focus 视觉语义。

预计会涉及以下区域：

- `domain/model/SmbEntry.kt`

## 验证与回归点

- 根目录与子目录都能正确展示列表行的名称、大小、最后修改时间。
- `..` 始终固定在最上方。
- 无论哪种排序方式，目录始终在文件前。
- 大小或时间缺失时显示 `--`，页面不崩溃。
- 排序切换后，焦点优先回到原先选中的同一路径项。
- 名称过长时，聚焦态能横向滚动，失焦后恢复省略。
- 刷新按钮在正常场景下刷新目录，在失败场景下也承担重试职责。
- 快速定位模式在排序切换后仍能正确进入、取消和接受。
- `assembleDebug` 必须通过，确保 XML、字符串和 Kotlin 代码没有引入编译错误。

## 实现建议

推荐按以下顺序实现：

1. 扩展 `SmbEntry` 和 SMB 仓库元数据采集。
2. 在 ViewModel 中加入排序状态与统一排序函数。
3. 调整顶部工具区，删除“重试连接”，加入“排序”按钮。
4. 实现轻量排序选择面板。
5. 扩展列表行布局，加入大小和时间两列，并处理名称聚焦跑马灯。
6. 验证焦点恢复、快速定位、刷新失败重试和编译打包。
