# TV 文件浏览长列表加速定位设计

- 日期：2026-04-06
- 适用范围：`TvBrowseFragment` 文件浏览页
- 设计目标：在保持 TV 遥控器低学习成本的前提下，提升超长文件列表的浏览效率与回访效率

## 1. 背景

当前文件浏览页面向 TV 遥控器优先，基础逐项焦点移动可用，但当目录内文件列表过长时，用户只能通过持续长按下键滚动。该交互在数百项甚至上千项目录中效率较低，且反复退出再进入同一目录后，用户需要重新寻找原来的浏览位置。

本设计仅解决两个高优先级问题：

1. 大步浏览：让用户在长列表中可以快速跨段移动，而不是只能逐项滚动。
2. 回访恢复：让用户重新进入同一目录时，可以尽可能回到上次浏览位置。

本设计不引入数字键、菜单键等额外硬件依赖，只使用：

- 方向键
- 确认键
- 返回键
- 长按确认键

## 2. 设计结论

采用 `A + B` 组合方案：

1. `A. 快速定位模式`
   用户在普通浏览态对列表项长按确认，进入“快速定位模式”。该模式临时把方向键语义从“逐项移动”切换为“整屏跳跃 + 分段跳跃”。
2. `B. 目录级锚点恢复`
   浏览页按目录持久化“最后焦点项”相关信息。用户再次进入同一目录时，若只有一个明确锚点则静默恢复；若存在多个强锚点冲突，则直接从头浏览，不弹窗、不要求用户做选择。

不纳入本次设计的内容：

- 常驻右侧索引条
- 多历史锚点面板
- 字母 / 拼音定位
- 智能学习式跳跃粒度

## 3. 交互设计

### 3.1 普通浏览态

默认仍保持现有列表浏览方式，不增加常驻额外 UI。

```text
文件浏览
┌─────────────────────────────────┐
│ 当前路径：/Albums/Jay/          │
│                                 │
│  Track 116.flac                 │
│  Track 117.flac                 │
│▶ Track 118.flac                 │
│  Track 119.flac                 │
│  Track 120.flac                 │
│                                 │
│  ↑/↓: 逐项移动焦点              │
│  确认: 打开 / 播放              │
│  长按确认: 进入快速定位模式      │
│  返回: 上级目录 / 退出           │
└─────────────────────────────────┘
```

规则：

1. 默认不显示比例条。
2. 默认不显示“快速定位模式”提示面板。
3. 用户只有在明确执行“长按确认”时，才进入增强定位能力。

### 3.2 快速定位模式

用户在普通浏览态长按确认进入该模式。进入后，显示临时比例条与模式说明；退出后立即恢复为普通浏览态。

```text
文件浏览
┌─────────────────────────────────┐   ┌──┐
│ 当前路径：/Albums/Jay/          │   │  │
│                                 │   │██│
│  Track 106.flac                 │   │  │
│▶ Track 118.flac                 │   │  │
│  Track 130.flac                 │   │  │
│                                 │   └──┘
│  快速定位模式  42%              │
│  ↑/↓: 整屏跳                    │
│  ←/→: 10% 分段跳                │
│  确认: 落到当前位置并退出模式     │
│  返回: 放弃本次跳跃并退出模式     │
└─────────────────────────────────┘
```

规则：

1. 比例条仅在快速定位模式内显示，普通浏览态完全隐藏。
2. `↑/↓` 执行整屏跳跃。
3. `←/→` 执行按列表总长度计算的固定比例跳跃，第一版使用 `10%`。
4. `确认` 接受当前落点，并退出快速定位模式。
5. `返回` 撤销本次跳跃，并退出快速定位模式。
6. 该模式不再弹额外面板，模式说明直接在列表底部就地展示。

设计理由：

- 不污染默认界面，保持普通浏览清爽。
- 复用 TV 最稳定的按键集合，避免依赖遥控器差异化功能。
- 把“逐项浏览”和“快速跳跃”明确拆为两种状态，降低误操作。

### 3.3 快速定位模式的进入限制

当目录内项目数较少时，进入快速定位模式收益很低，反而增加理解成本。建议设置进入门槛：

1. 当列表项数小于一个整屏容量的 2 倍时，不进入快速定位模式。
2. 若用户长按确认但不满足门槛，可短暂提示“当前目录无需快速定位”。

该门槛避免用户在短列表上误入模式。

## 4. 锚点恢复设计

### 4.1 锚点定义

每个目录维护一个轻量浏览锚点，记录：

- 目录路径
- 最后焦点项稳定标识
- 最后焦点项索引
- 写入时间

此外，系统已有“定位当前播放目录”能力，播放相关路径也可作为辅助参考，但不直接形成新的交互面板。

### 4.2 恢复策略

重新进入目录时，按以下规则处理：

1. 若只有一个明确锚点，则静默恢复到该位置。
2. 恢复后显示 1 到 2 秒轻提示，例如“已恢复上次位置”。
3. 若存在多个强锚点冲突，例如“上次焦点位置”和“最近播放位置”明显不同，则不弹窗、不让用户选择，直接从头浏览。
4. 若锚点项不存在，则先尝试恢复到相邻索引；索引也失效时，退化到目录开头。
5. “从头浏览”是所有冲突和失败场景的统一兜底策略。

统一原则：

- 能明确恢复就恢复
- 一旦不确定就从头
- 不增加额外交互步骤

### 4.3 不采用锚点选择弹窗

本设计明确不采用“上次位置 / 最近播放 / 从头浏览”的选择弹窗。

原因：

1. TV 遥控器交互成本高，弹窗意味着额外焦点流转与额外确认动作。
2. 用户在进入目录这一基础动作上更需要稳定预期，而不是面对条件判断。
3. “不确定就从头”的规则比“弹给用户选”更简单、更一致。

## 5. 数据与持久化

### 5.1 持久化内容

建议新增浏览锚点存储，按“连接 + 目录路径”维度保存轻量状态。

建议字段：

- `connectionId` 或可匹配连接标识
- `directoryPath`
- `lastFocusedItemKey`
- `lastFocusedIndex`
- `updatedAt`

其中：

- `lastFocusedItemKey` 优先使用能稳定匹配目录项的唯一值
- `lastFocusedIndex` 作为降级恢复依据

### 5.2 与清除缓存的关系

设置页已有“清除缓存”能力。浏览锚点属于缓存型持久化状态，必须受该按钮统一控制。

要求：

1. 用户执行“清除缓存”时，浏览锚点数据一并清空。
2. 清空后再次进入任何目录，都应从目录开头开始浏览。
3. “清除缓存”后不保留上次焦点、上次索引、目录恢复提示等历史信息。

该规则确保“清除缓存”语义完整，避免用户误以为缓存已清但浏览状态仍被记住。

## 6. 组件边界

### 6.1 浏览 UI 层

`TvBrowseFragment` 负责：

- 处理普通浏览与快速定位模式的 UI 切换
- 响应长按确认、方向键、返回键
- 显示或隐藏比例条与模式提示
- 在列表渲染完成后请求正确焦点

### 6.2 浏览状态层

`TvBrowserViewModel` 负责：

- 保存当前模式状态
- 计算快速跳跃后的目标索引
- 协调目录进入、目录返回、播放定位后的恢复逻辑
- 写入与读取目录锚点

### 6.3 持久化层

建议新增或扩展现有浏览状态存储，负责：

- 保存目录锚点
- 按目录读取锚点
- 在“清除缓存”时清空锚点

该层不负责 UI 策略判断，只提供数据读写能力。

## 7. 数据流

### 7.1 快速定位模式

1. 用户在普通浏览态长按确认。
2. UI 层检查当前目录项目数是否达到快速定位门槛。
3. 若满足门槛，则进入快速定位模式并显示比例条。
4. 用户使用方向键执行跳跃，ViewModel 计算目标索引。
5. UI 将列表滚动并把焦点落在目标项。
6. 用户按确认接受落点，或按返回取消跳跃，随后退出模式。

### 7.2 目录锚点恢复

1. 用户进入某目录。
2. ViewModel 读取当前目录锚点与辅助参考信息。
3. 若只有一个明确锚点，则列表加载完成后恢复焦点。
4. 若存在冲突或恢复失败，则直接从头浏览。
5. 用户在目录内移动焦点、退出目录或进入下一级目录前，更新当前目录锚点。

### 7.3 清除缓存

1. 用户在设置页点击“清除缓存”。
2. 设置页调用统一缓存清理入口。
3. 浏览锚点存储被清空。
4. 后续任何目录进入操作均不再尝试恢复旧锚点。

## 8. 异常处理

需要覆盖以下异常场景：

1. 目录内容变化，历史焦点项已被删除或重命名。
2. 历史索引超出当前列表长度。
3. 列表过短，不值得进入快速定位模式。
4. 长按确认阈值不稳定导致误触发。
5. 目录重新加载后，焦点请求发生在视图尚未完成布局前。

处理原则：

1. 优先安全回退到目录开头。
2. 不展示错误弹窗。
3. 允许短提示，但不要求用户额外确认。

## 9. 测试与验收

### 9.1 核心验收项

1. 20 项、200 项、2000 项目录均可正常浏览。
2. 长按确认只在长列表中进入快速定位模式。
3. 快速定位模式下的比例条默认隐藏，仅在模式内出现。
4. 快速定位模式下，`↑/↓`、`←/→`、`确认`、`返回` 的语义符合设计。
5. 从子目录返回父目录时，可恢复到原焦点位置。
6. 从播放页定位回目录时，与浏览恢复逻辑兼容，不出现异常焦点。
7. 历史锚点失效时，能稳定退回目录开头。
8. 执行“清除缓存”后，所有目录恢复信息都被重置。

### 9.2 回归关注点

1. 不影响现有目录加载、点击进入、播放单曲、播放整个目录等功能。
2. 不破坏现有“定位当前播放目录”流程。
3. 不引入新的焦点丢失、空白焦点或错误滚动问题。

## 10. 实施建议

建议按以下顺序实施：

1. 抽离目录锚点数据结构与持久化接口。
2. 接入“清除缓存”联动，确保锚点可被统一重置。
3. 实现列表焦点恢复逻辑。
4. 实现快速定位模式状态机与按键处理。
5. 打通进入目录、返回目录、播放定位目录三类路径的统一恢复逻辑。
6. 完成回归验证与真机调试。

## 11. 非目标

以下内容不属于本轮设计范围：

1. 全库搜索
2. 文件名筛选
3. 字母索引 / 拼音索引
4. 常驻右侧进度索引条
5. 多级历史记录面板

## 12. 设计摘要

本设计坚持两个原则：

1. 增强能力必须按需出现，不能污染默认浏览态。
2. 恢复策略必须简单可预测，不能把判断成本转嫁给用户。

因此，最终方案是：

- 默认普通浏览
- 长按确认进入快速定位模式
- 比例条只在模式内显示
- 有单一明确锚点时静默恢复
- 一旦锚点冲突或失效，直接从头浏览
- “清除缓存”会同步清空所有锚点持久化信息
