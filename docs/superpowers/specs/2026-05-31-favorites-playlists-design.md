# 收藏与播放列表功能设计

## 背景

TSM Player 当前支持 SMB 浏览、目录播放、播放页控制、最近播放恢复和播放定位，但还没有长期收藏/歌单能力。本次增加收藏功能，目标是在播放页快速收藏当前歌曲，在列表页进入收藏管理页，并支持默认收藏夹、自建播放列表、播放和移除歌曲。

## 范围

本次实现包含：

- 播放页底部按钮栏新增收藏按钮，位于播放模式按钮右侧。
- 播放页收藏按钮短按加入默认收藏夹，长按弹出选择播放列表 modal，并支持新建播放列表。
- 列表页播放控制区域在顺序播放左侧新增红色收藏入口按钮，点击进入收藏页。
- 列表页收藏、顺序播放、随机播放三个按钮改为默认仅显示 SVG 图标，聚焦时显示图标和文字；顺序播放、随机播放复用播放页播放模式相关 SVG。
- 收藏页使用宫格样式展示播放列表：左上角为添加播放列表，其后为默认收藏夹和自定义播放列表。
- 播放列表宫格封面显示该列表最后加入歌曲的封面；没有封面时使用默认封面。
- 点击播放列表后进入该列表的歌曲纵向列表；歌曲支持点击播放，右侧删除 SVG 用于从列表移除，不删除 SMB 原文件。
- 收藏数据使用 SQLite 持久化，方便后续迁移和扩展。

不包含：

- 云同步、多设备同步。
- 歌单重命名、排序、批量移动。
- 播放页一键取消收藏状态。
- 删除 SMB 原始文件。

## UI 设计

### 播放页

收藏按钮放在播放模式按钮右侧，短按和长按语义都围绕当前播放歌曲。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [上一] [播放] [下一] [播放模式] [收藏] [睡眠] [歌词全屏] [返回] [定位]       │
│                         默认: 红底白心形图标                                │
│                         聚焦: 红底白心形图标 + 收藏                         │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 使用 `@style/TsmButtonDanger` 或同等红色按钮样式，颜色来自 `@color/ui_accent_red*`。
- 默认折叠宽度复用播放页紧凑按钮尺寸。
- 获得焦点时展开最小宽度，显示图标和“收藏”。
- `nextFocusUp` 指向播放进度条。

### 列表页播放控制区

列表页播放控制区保持一行布局，收藏入口在顺序播放左侧，回到当前播放按钮占用剩余宽度。

```text
播放控制
┌──────────────────────────────────────────────────────────────────────────────┐
│ [❤] [▶] [🔀] [回到当前播放：歌曲标题......................................] │
│  │   │    │   └─ 自适应剩余宽度，长文本走马灯                               │
│  │   │    └─ 聚焦显示：随机播放                                             │
│  │   └─ 聚焦显示：顺序播放，复用播放页顺序播放 SVG                          │
│  └─ 聚焦显示：收藏，点击进入收藏页                                          │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 三个入口按钮默认只显示 SVG 图标，聚焦显示图标和文字。
- 顺序播放使用 `ic_play_order`。
- 随机播放使用 `ic_shuffle`。
- 收藏使用新增心形收藏 SVG。
- 回到当前播放/继续上次播放按钮仍保持 warning 样式，并自适应剩余宽度。

### 收藏页：播放列表宫格

收藏页是独立 Activity。首页以宫格展示播放列表。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 收藏                                                                         │
│                                                                              │
│ ┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐                 │
│ │     +      │ │  最后封面  │ │  最后封面  │ │  最后封面  │                 │
│ │ 添加列表   │ │  收藏夹    │ │  歌单 A    │ │  歌单 B    │                 │
│ └────────────┘ └────────────┘ └────────────┘ └────────────┘                 │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 第一个宫格固定为“添加播放列表”。
- 第二个宫格固定为默认播放列表“收藏夹”。
- 后续宫格为用户创建的播放列表。
- 宫格封面使用该列表最后加入歌曲的封面 URI；无封面时使用 `default_cover`。
- 宫格焦点使用项目现有蓝色焦点描边和深色面板风格，不新增通用硬编码颜色。

### 收藏页：歌曲列表

点击某个播放列表后，进入该列表的歌曲纵向列表。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 收藏夹                                                       [返回列表宫格]  │
│                                                                              │
│ 01  歌曲标题 - 艺术家                                      [播放] [删除svg] │
│ 02  歌曲标题 - 艺术家                                      [播放] [删除svg] │
│ 03  歌曲标题 - 艺术家                                      [播放] [删除svg] │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 歌曲行采用类似列表页纵向展开体验。
- 点击歌曲行或播放按钮，从当前播放列表构建队列，并从该歌曲开始播放。
- 删除 SVG 只移除收藏记录，不删除 SMB 文件。
- 删除默认收藏夹中的歌曲也是移除记录，不删除默认列表本身。

## 交互设计

### 播放页收藏

- 当前没有可收藏歌曲时：
  - 短按或长按收藏按钮显示轻提示“暂无可收藏歌曲”。
- 短按 OK：
  - 将当前歌曲加入默认收藏夹。
  - 若已存在，提示“已在收藏夹中”。
  - 成功后提示“已加入收藏夹”。
- 长按 OK：
  - 弹出选择播放列表 modal。
  - 列表包含“收藏夹”、所有自建播放列表和“新建播放列表”。
  - 选择现有播放列表后加入该列表。
  - 选择新建播放列表后弹输入框；创建成功后将当前歌曲加入新列表。

### 列表页收藏入口

- 点击收藏入口打开收藏页。
- 该入口不收藏当前目录或当前文件，避免与文件焦点语义冲突。

### 收藏页播放

- 从某个播放列表播放时，将列表中的歌曲按当前展示顺序构建 Media3 队列。
- 点击某首歌曲时，从该歌曲开始播放。
- 如果歌曲 URI 已失效，沿用现有播放失败 Toast 风格提示。

## 数据模型

使用 SQLite，初始版本为 `1`。

### playlists

```text
id TEXT PRIMARY KEY
name TEXT NOT NULL
is_default INTEGER NOT NULL
created_at INTEGER NOT NULL
updated_at INTEGER NOT NULL
```

- 默认收藏夹固定 `id = "default_favorites"`，`name = "收藏夹"`，`is_default = 1`。
- 自定义播放列表使用 UUID 字符串。

### playlist_tracks

```text
id TEXT PRIMARY KEY
playlist_id TEXT NOT NULL
media_id TEXT NOT NULL
stream_uri TEXT NOT NULL
title TEXT NOT NULL
artist TEXT
album TEXT
artwork_uri TEXT
source_connection_id TEXT
source_host TEXT
source_share TEXT
source_path TEXT
source_username TEXT
source_password TEXT
source_guest INTEGER
source_smb1 INTEGER
added_at INTEGER NOT NULL
UNIQUE(playlist_id, media_id)
```

- `media_id` 采用当前 MediaItem 的 `mediaId`，通常等于 SMB `fullPath`。
- `stream_uri` 用于直接恢复播放。
- 标题、艺术家、专辑、封面 URI 保存加入时的展示信息，后续可迁移到更完整的标签表。
- 保存来源连接配置，便于后续定位、恢复和迁移。
- 同一播放列表内同一 `media_id` 不重复加入。

## 架构

新增 `favorites` 包：

```text
favorites/
├─ FavoritePlaylist.kt
├─ FavoriteTrack.kt
├─ FavoritesDbHelper.kt
└─ FavoritesRepository.kt
```

- `FavoritesDbHelper` 负责 SQLite 建表、升级和默认收藏夹初始化。
- `FavoritesRepository` 提供播放列表和歌曲的增删查接口。
- UI 层不直接拼 SQL。
- 播放页、列表页和收藏页共享同一个 repository。

新增 `FavoritesActivity`：

- 宫格模式展示播放列表。
- 列表模式展示某个播放列表内歌曲。
- 负责新建播放列表、移除歌曲、构建播放队列并启动播放。

## 错误处理

- 新建播放列表名称为空：提示“请输入播放列表名称”。
- 新建播放列表重名：提示“播放列表已存在”。
- 收藏当前歌曲时无播放内容：提示“暂无可收藏歌曲”。
- 重复收藏：提示“已在该播放列表中”。
- 播放列表为空：提示“该播放列表暂无歌曲”。
- 播放失败：沿用现有 SMB 播放失败文案风格。

## 测试与验证

最低验证：

- `.\gradlew.bat testDebugUnitTest`
- `.\gradlew.bat assembleDebug`

手动验证：

- 播放页短按收藏按钮加入默认收藏夹。
- 播放页长按收藏按钮选择已有列表和新建列表。
- 列表页收藏入口打开收藏页。
- 收藏页宫格顺序为添加、收藏夹、自定义播放列表。
- 歌曲加入后对应播放列表封面变为最后加入歌曲封面。
- 点击收藏歌曲可播放，队列顺序正确。
- 点击删除 SVG 后只从列表移除，不删除 SMB 原文件。
- 全局缩放 90% / 100% / 110% 下按钮不重叠，回到当前播放按钮仍自适应剩余宽度。
