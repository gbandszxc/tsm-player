# TV 自绘 Modal 统一设计

## 背景

TSM Player 当前在播放页、收藏页、SMB 浏览页、设置页、更新检测和启动提示中仍混用 Android 原生 `AlertDialog` / `ProgressDialog`。这些系统框体与项目现有的 TV 深色面板、蓝色焦点、分层按钮语义和遥控器焦点链风格不一致，导致同一应用内出现明显割裂。

本次设计目标是统一所有用户可见的弹层体验，形成一套可复用的 TV 自绘 modal 体系，用于替换原生框体，同时保持当前项目的设计 token、焦点规则和低干扰浏览/播放体验。

## 目标

- 用统一的自绘 TV modal 替换现有用户可见的 Android 原生对话框与原生下载进度框。
- 保持与 `DESIGN.md` 一致的深色面板、蓝色焦点、绿/黄/红语义按钮与 MiSans 字体层级。
- 保证遥控器 D-Pad、确认键、返回键在所有 modal 中行为一致且可预测。
- 抽出一套公共 modal 基础能力，避免 `Activity` / `Fragment` / 管理器类各自维护不同实现。

## 非目标

- 不替换系统安装页面、系统权限页面本身。
- 不改 Toast 体系。
- 不在本轮扩展到非弹层页面结构重做。
- 不引入与项目现有 XML/UI 模式冲突的新 UI 技术栈。

## 统一设计方向

采用“`A` 为主、吸收 `B` 局部表达”的方向：

- 外层统一为居中 TV modal 面板，带深色遮罩。
- modal 内部统一分为标题区、内容区、操作区。
- 连接管理类弹层在内容区吸收少量“命令面板”表达，强化 TV 系统工具感。
- 其他场景保持列表式、表单式、确认式的高一致骨架。

## 统一视觉语言

- 遮罩：复用 `@color/ui_bg_overlay`，让底层页面仍可感知但不抢焦点。
- 面板：复用 `@color/ui_bg_panel` / `@color/ui_bg_panel_deep`，圆角使用 `@dimen/ui_radius_panel`。
- 分隔线：复用 `@color/ui_divider`。
- 标题：主标题使用 `@style/TsmTextSectionTitle` 或同级字重，辅助说明使用 `@style/TsmTextSecondary`。
- 焦点：聚焦项使用亮蓝描边与更亮底色，不能仅靠颜色变化表达。
- 按钮：
  - 主操作：`@style/TsmButtonPrimary`
  - 成功/继续：`@style/TsmButtonSuccess`
  - 提醒/次重要操作：`@style/TsmButtonWarning` 或 `@style/TsmButtonDark`
  - 危险操作：`@style/TsmButtonDanger`
- 表单错误：在输入框附近直接展示错误文案，同时保留 Toast 作为补充，不只靠红色或系统错误气泡。

## 公共 Modal 骨架

统一 modal 结构：

```text
┌──────────────────────────────────────────────────────────────┐
│ 分类标签                                                     │
│ 主标题                                              返回/关闭 │
├──────────────────────────────────────────────────────────────┤
│ 说明文案 / 当前上下文                                         │
│                                                              │
│ [命令项 / 列表项 / 输入项 / 确认内容]                         │
│ [命令项 / 列表项 / 输入项 / 确认内容]                         │
│ [命令项 / 列表项 / 输入项 / 确认内容]                         │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│                         [次按钮] [主按钮]                     │
└──────────────────────────────────────────────────────────────┘
```

通用规则：

- modal 打开后，初始焦点必须落在第一个可操作控件上。
- `Back` 关闭 modal 并回到打开前的焦点。
- 遮罩层不可聚焦，不拦截为默认焦点。
- 内容区超高时仅内容区滚动，标题区与操作区保持稳定。
- 表单 modal 在输入报错时，不自动关闭，焦点留在错误项或主操作按钮附近。

## Modal 类型

### 1. ActionModal

适用于：连接管理、睡眠权限提示、发现新版本。

特点：

- 以说明 + 2~3 个动作按钮为主。
- 连接管理场景在内容区以“命令面板”样式展示动作项，而不是底部三键平均分配。
- 更新提示和权限提示保持短文案 + 明确主次操作。

### 2. ListModal

适用于：切换连接、选择播放列表。

特点：

- 使用纵向可聚焦列表。
- 当前选项、不可用选项和“创建新项”在视觉上明确区分。
- 不可用项不能像普通项一样被误触发；如允许聚焦，需要明确展示“已存在/不可选”原因。

### 3. FormModal

适用于：新建播放列表、SMB 连接配置、歌词字号输入、歌词间距输入。

特点：

- 标题区下方先给一行说明。
- 表单项采用统一 label + 输入区布局，不复用 Android 原生输入框外观。
- SMB 配置使用可滚动表单，兼容小屏 TV 和软键盘遮挡。
- 错误信息内嵌在表单中，不关闭 modal。

### 4. ConfirmModal

适用于：失效歌曲是否移除。

特点：

- 信息密度低，强调操作后果。
- 主按钮危险色，次按钮暗色。
- 文案明确“只会移除收藏记录，不删除 SMB 原文件”。

### 5. ProgressModal

适用于：更新下载进度。

特点：

- 替换原生 `ProgressDialog`。
- 以 modal 面板承载文件名、百分比、进度条和状态文案。
- 下载期间默认不可取消，避免半下载状态引发误解。

## 页面映射

本轮统一纳入以下入口：

1. 播放页 `PlaybackActivity`
- 长按收藏 -> `ListModal`：选择播放列表
- 从选择列表进入新建 -> `FormModal`：新建播放列表
- 失效歌曲播放时 -> `ConfirmModal`：是否移除

2. 收藏页 `FavoritesActivity`
- 添加播放列表 -> `FormModal`：新建播放列表
- 失效歌曲播放时 -> `ConfirmModal`：是否移除

3. SMB 浏览页 `TvBrowseFragment`
- 连接管理 -> `ActionModal`
- 切换连接 -> `ListModal`
- 编辑当前连接 / 新建连接 -> `FormModal`：SMB 连接配置

4. 首页 `MainActivity`
- 首次启动睡眠权限提示 -> `ActionModal`

5. 更新检测 `AppUpdateManager`
- 发现新版本 -> `ActionModal`
- 下载更新中 -> `ProgressModal`

6. 设置页 `SettingsActivity`
- 歌词字号输入 -> `FormModal`
- 歌词间距输入 -> `FormModal`

## 关键交互设计

### 连接管理

连接管理 modal 采用居中面板，但内容区吸收少量分栏命令感：

```text
┌──────────────────────────────────────────────────────────────┐
│ SMB                                                          │
│ 连接管理                                                返回  │
├──────────────────────────────────────────────────────────────┤
│ 当前连接：smb://192.168.0.10（全部共享）                     │
│                                                              │
│ [编辑当前连接]                                               │
│ [新建连接]                                                   │
│ [切换连接]                                                   │
└──────────────────────────────────────────────────────────────┘
```

规则：

- 初始焦点落在“编辑当前连接”；若当前无可编辑连接，则落在“新建连接”。
- “切换连接”进入连接列表 modal，不在同框内塞入二级列表，避免焦点链复杂化。

### 切换连接

```text
┌──────────────────────────────────────────────────────────────┐
│ SMB                                                          │
│ 切换连接                                                返回  │
├──────────────────────────────────────────────────────────────┤
│ 请选择要切换的已保存连接                                      │
│ > 客厅 NAS / 192.168.0.10                                    │
│   工作室 NAS / 192.168.0.12                                  │
│   备用 NAS / 192.168.0.20                                    │
├──────────────────────────────────────────────────────────────┤
│                                 [取消] [连接到所选连接]       │
└──────────────────────────────────────────────────────────────┘
```

规则：

- 列表按视觉顺序聚焦。
- 确认键可直接切换，底部主按钮作为冗余可见动作。
- 空列表不弹 modal，继续保留现有 Toast 提示。

### 选择播放列表

```text
┌──────────────────────────────────────────────────────────────┐
│ 收藏                                                         │
│ 选择播放列表                                            返回  │
├──────────────────────────────────────────────────────────────┤
│ 将当前歌曲加入以下播放列表                                    │
│ > 收藏夹                                                     │
│   夜间播放                                                   │
│   通勤                                                       │
│   + 新建播放列表                                             │
│   工作 BGM（已存在）                                         │
└──────────────────────────────────────────────────────────────┘
```

规则：

- “新建播放列表”作为与普通项同级的最后一项。
- 已存在项默认可聚焦但显示禁用原因；确认时只提示“已在该播放列表中”，不关闭。
- 若用户更偏好更直接，也可在实现中让已存在项不可聚焦，但需要保留明显的禁用视觉。

### 新建播放列表

```text
┌──────────────────────────────────────────────────────────────┐
│ 收藏                                                         │
│ 新建播放列表                                            返回  │
├──────────────────────────────────────────────────────────────┤
│ 名称                                                        │
│ [ 我的歌单______________________________________________ ]   │
│ 错误文案：播放列表已存在                                      │
├──────────────────────────────────────────────────────────────┤
│                                     [取消] [创建并加入/创建]  │
└──────────────────────────────────────────────────────────────┘
```

规则：

- 在播放页进入时，主按钮文案为“创建并加入”。
- 在收藏页入口进入时，主按钮文案为“创建”。
- 名称为空或重复时，modal 不关闭，错误文案就地展示。

## 架构设计

采用公共 modal 体系，而不是每个页面各写一套弹窗：

- 提供统一的 modal 容器、标题区、内容区、操作区、遮罩和焦点恢复。
- 在其上封装 5 类模板：`ActionModal`、`ListModal`、`FormModal`、`ConfirmModal`、`ProgressModal`。
- 由页面传入文案、字段定义、列表项、按钮语义和回调。
- 统一处理：
  - 显示/关闭
  - 初始焦点
  - Back 关闭
  - 焦点恢复
  - 表单错误展示
  - 列表滚动与可见区域

实现层面优先采用“自定义 Dialog + 公共内容 View”的轻量方案，而不是每个页面自写或一次性重构到 `DialogFragment`。

原因：

- 适配当前项目中 `Activity`、`Fragment`、更新管理器等多处调用点更直接。
- 能快速统一样式，同时保留良好的封装边界。
- 后续若需要，再逐步演化到 `DialogFragment`，成本更可控。

## 焦点与输入规则

- modal 打开时缓存调用前焦点，关闭时恢复。
- 遥控器上下键按视觉顺序移动；左右键优先在同一行的输入/按钮间切换。
- 底部操作区始终在内容区之后，避免形成焦点环路。
- 表单 modal 中，输入框聚焦态必须明显区别于非焦点态。
- 输入校验失败时：
  - 展示就地错误文案
  - 可附带 Toast
  - 不关闭 modal
  - 焦点移动到首个错误字段或保留在确认按钮以便重试

## 错误处理

- 列表为空但理论上不应弹框的场景，继续使用 Toast 并不显示空 modal。
- 更新下载失败时关闭进度 modal，并用 Toast 说明原因。
- 保存连接失败、创建歌单失败等校验型错误优先在 modal 内表达。
- 连接/网络型错误仍可由页面级状态或 Toast 兜底，不把复杂错误都塞进 modal。

## 测试与验收

需要覆盖以下验收点：

1. 视觉一致性
- 所有 modal 复用项目 token，无新增通用硬编码颜色、圆角、字号。
- 与播放页、收藏页、设置页现有控件风格一致。

2. 遥控器交互
- 初始焦点正确。
- `Back` 正常关闭并恢复原焦点。
- 列表 modal 可完整上下导航。
- 表单 modal 的输入和底部操作可稳定切换。

3. 业务行为
- 播放页新建歌单后可直接加入当前歌曲。
- 收藏页新建歌单后刷新宫格。
- 切换连接成功触发既有 ViewModel 行为。
- 失效歌曲移除只删除收藏记录。
- 更新下载进度正确显示并在完成后进入安装。

4. 回归
- 不影响现有 Toast、页面导航、Media3 播放、SMB 配置保存与更新安装流程。

## 文档维护要求

实现完成后，需同步维护：

- `DESIGN.md`：补充“公共 TV modal 体系”与各类 modal 的最终交互规范。
- `docs/MANUAL.md`：记录原生框替换范围、统一组件名称、下载进度框替换与相关交互更新。

## 实施边界

该设计足够支持单次实现计划，无需再拆成多个独立 spec。实现可分阶段提交，但应以“公共 modal 体系先落地，再逐步替换各入口”为主线，避免中途出现新的第三套弹窗风格。
