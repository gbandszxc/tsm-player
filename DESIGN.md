# Android TV UI/交互设计规范

> 最后同步：2026-06-05，基于 SMB 连接配置回显优化后的实现状态。

## Overview

TSM Player 是面向 Android TV 的 SMB 本地音乐播放器。界面优先服务遥控器操作、远距离阅读、稳定焦点反馈和低干扰播放体验。

核心体验目标：

- 用户坐在电视距离外也能快速判断当前焦点、当前页面、播放状态和可执行操作。
- 遥控器焦点始终稳定、可预测，不被装饰性元素或不可操作内容抢走。
- 播放与浏览流程保持低干扰：设置、缓存、继续播放和定位能力都围绕“少打断、可恢复”设计。
- 设计规范既服务人类阅读，也通过 YAML front matter 为后续工具和代理提供机器可读 token。

## Colors

颜色以深色背景、蓝色焦点、绿色播放操作和黄色提醒操作组成。Android XML 使用 `@color/ui_*`，不得新增通用硬编码颜色。

- 背景层：`ui_bg_app` 用于主深色页面，`ui_bg_app_deep` 用于更沉浸的全屏歌词，`ui_bg_sidebar` 用于设置页左栏，`ui_bg_panel*` 用于面板、卡片和设置项。
- 文本层：`ui_text_primary` 用于主文本，`ui_text_secondary` 用于说明文本，`ui_text_muted` 用于弱信息和非当前歌词。
- 蓝色层：`ui_accent_blue*` 用于页面标题、遥控器焦点、选中指示条和主操作按钮。
- 绿色层：`ui_accent_green*` 用于播放、继续播放之外的成功/媒体操作。
- 黄色层：`ui_accent_yellow*` 和 `ui_accent_amber*` 用于“定位”“继续上次播放”等需要用户注意但不表示危险的操作。
- 标签层：目录标签使用 `ui_tag_dir`，音频标签使用 `ui_tag_audio`。

## Typography

字体使用 MiSans。Android XML 字号使用 `@dimen/ui_text_*`，页面标题、分区标题、正文和辅助文本保持清晰层级。

- 页面大标题使用 `ui_text_screen_title`，例如设置页左栏“设置”。
- 内容区标题使用 `ui_text_page_title`，例如设置页右侧当前分类标题。
- 列表项和按钮主文本使用 `ui_text_title` 或 `ui_text_body_large`。
- 辅助说明、标签、缓存大小、歌词弱信息使用 `ui_text_body` 或 `ui_text_caption`。
- 歌词字号可由设置页单独配置，遵守 `UiSettingsStore` 中的可调范围。

## Layout

页面按 TV 横屏优先设计，内容区使用稳定尺寸、明确焦点链和可滚动容器。布局值优先使用 `@dimen/ui_space_*`；具备行为意义的尺寸可以保留专用值，例如设置页左栏宽度、封面比例容器和列表项稳定高度。

- 首页为纵向滚动页面，连接管理、播放控制和文件浏览按任务优先级分组。
- 播放页上半区为封面和歌曲信息/歌词，下半区为遥控器可达的按钮栏。
- 播放页底部按钮栏顺序为上一首、播放/暂停、下一首、播放模式、收藏、睡眠定时、歌词全屏、返回文件页、定位；收藏按钮使用红色 danger 样式，默认仅显示爱心图标，聚焦展开为“收藏”。
- 设置页为左侧分类 + 右侧详情，左侧分类负责导航，右侧详情承载当前分类内容。
- 全局缩放支持 90% / 95% / 100% / 105% / 110%，布局必须避免在这些档位下出现文字遮挡或控件抖动。

## Elevation & Depth

项目不使用复杂投影层级。深浅背景、描边、焦点框和浮层透明背景承担视觉层次。

- 主页面背景最深，面板略亮，焦点项通过更亮底色与描边浮出。
- 快速定位模式使用右下角悬浮层，避免跟随长列表滚动。
- 图片全屏预览使用背景模糊图层 + 前景 `fitCenter` 原图形成深度，不裁切原图主体。

## Shapes

圆角使用 `@dimen/ui_radius_*`；常用规则为无圆角 0dp、标签 3dp、列表项 4dp、按钮 6dp、面板 8dp、选中指示条 2dp。

- 按钮圆角保持克制，避免 TV 工具界面变成过度装饰的卡片风格。
- 设置页当前分类不使用整框表达，使用左侧竖线 + 低亮底色，与遥控器焦点框分离。
- 封面焦点框使用直角或接近直角的清晰描边，优先表达“可按确认进入预览”。

## Components

### 公共样式类

- `@style/TsmTextPrimary`：主文本。
- `@style/TsmTextSecondary`：辅助文本。
- `@style/TsmTextPageTitle`：页面大标题。
- `@style/TsmTextSectionTitle`：内容区标题。
- `@style/TsmButtonPrimary`：主操作按钮。
- `@style/TsmButtonDark`：低强调操作按钮。
- `@style/TsmButtonSuccess`：成功/播放类操作按钮。
- `@style/TsmButtonWarning`：恢复、继续播放、定位等提醒型操作按钮。
- `@style/TsmButtonDanger`：关闭睡眠等表示不可逆/停止操作的红色按钮。

新增 XML UI 样式必须优先引用 `@color/ui_*`、`@dimen/ui_*`、`@style/Tsm*` 和既有 `@drawable/bg_*`，不得新增散落的通用硬编码颜色或圆角。

- 播放控制中的收藏、顺序播放、随机播放为紧凑图标按钮，未聚焦时只显示图标；获得焦点时展开为图标 + 文案。收藏按钮使用 `TsmButtonDanger`，点击进入收藏页。
- SMB 浏览页收藏、顺序播放、随机播放三枚紧凑按钮在未聚焦态必须和播放页底部按钮栏保持同一套布局逻辑：矩形宽度固定、图标通过 overlay 视觉居中，不能因为 `drawableStart` 默认排版而向左偏移。
- 黄色按钮与绿色按钮同行，文字过长时走马灯滚动；仅在有活跃播放或有上次播放记录且开关开启时显示。
- 文件浏览顶部工具区收敛为"刷新当前目录"和"排序"两个可聚焦控件；排序控件按 OK 展开轻量锚定式下拉面板，选中后立即收起并应用；当面板下方可见空间不足时，页面自动下滚到能完整显示所有排序选项。
- SMB 浏览页进入当前连接时会自动尝试连接当前目录，网络类失败按指数退避最多尝试 3 次；仍失败时显示 Toast 和页面内错误状态，并锁定当前连接的自动目录刷新。锁定后，路径恢复、目录切换和定位等自动加载路径不再继续访问 SMB，直到用户手动点击"刷新当前目录"；每点击一次只触发一次新的 3 次退避重试流程。失败锁定仅保存在当前应用进程内，退出软件后重置。
- 排序下拉面板展开期间，上下方向键焦点限制在面板选项内，最后一项继续向下不会跳入文件列表。
- 触屏模式下，排序下拉面板打开后点击面板外区域只关闭下拉面板，不触发重排序，也不透传点击到背后的列表或按钮。
- 文件行显示类型标签、名称、大小、修改时间四列；大小和时间列固定宽度，真实值右对齐，缺省占位 `--` 居中显示，名称列聚焦时跑马灯滚动。目录行大小列显示 `--`，`..` 返回上一级行不显示大小与时间列。
- 默认按文件名升序排序，目录始终位于文件前，`..` 始终固定最上。排序选项：文件名 ↑/↓、文件大小 ↑/↓、修改时间 ↑/↓。
- 长列表支持长按确认进入快速定位模式；模式提示固定在根布局右下角悬浮显示，不随列表滚动。
- 目录级焦点锚点会按连接和路径保存，返回同一路径时恢复最近焦点。

### 连接管理弹窗

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 连接管理                                                                     │
├──────────────────────────────────────────────────────────────────────────────┤
│ 请选择操作                                                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│ [编辑当前连接]   [删除当前连接]   [新建连接]   [切换连接]                   │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 弹窗按钮保持横向扫描顺序。
- “删除当前连接”仅在当前激活连接来自已保存连接时显示，按钮使用 danger 语义，并在执行前弹出确认 modal。
- 点击“编辑当前连接”“新建连接”“切换连接”或“删除当前连接”后，连接管理弹窗应立即关闭，后续表单、列表或确认弹窗成为唯一前台上下文，避免保存、切换或删除完成后仍回到旧连接管理页。
- 删除最后一个已保存 SMB 连接后，首页回到未配置状态，并显示根目录连接管理区，用户可直接进入“新建连接”。
- Back 关闭弹窗并回到原页面焦点上下文。

- 上半区水平布局，封面 weight=3，信息区 weight=7。
- 封面用 `SquareFrameLayout` 保证 1:1，常规显示为 `centerCrop`。
- 播放页默认焦点停在进度条；进度条按上可聚焦封面图。
- 上一首、播放/暂停、下一首使用纯 VectorDrawable 图标按钮，聚焦时仍保持 icon-only，不展开文字；图标分别采用标准“上一曲竖线+三角”、“播放三角/暂停双竖条”和“下一曲三角+竖线”语义。
- 播放模式按钮位于”下一首”和”收藏”之间，默认顺序播放；OK 按顺序切换”顺序播放 / 单曲循环 / 列表循环 / 随机播放”。按钮使用播放中”暂停”同系的黄色/琥珀色，与上一首/下一首暗色按钮区分；未聚焦时为 icon 居中的紧凑矩形，获得焦点时展开显示短文字。
- 收藏按钮位于”播放模式”和”睡眠定时”之间，使用红色 danger 样式。未聚焦时为爱心图标紧凑矩形，获得焦点时展开显示“收藏”；短按加入/移出默认收藏夹，长按进入播放列表选择弹窗。
- 睡眠定时按钮位于”收藏”和”歌词全屏”之间，使用暗色背景（`TsmButtonDark`），与上一首/下一首按钮同系。未开启时显示空心月亮图标（居中紧凑）；开启后显示实心月亮图标与剩余分钟数字，分钟向上取整，宽度自动扩展。点击按钮进入睡眠定时设置页。

### 遥控器与焦点规则

- D-Pad 上下左右用于在当前页面可聚焦控件间移动；OK/确认用于进入目录、播放曲目、切换设置或触发按钮。
- Back 在浏览页优先返回上级目录，在设置页退出设置，在播放页返回上一层页面；全屏图片预览中 Back 退出预览。
- Menu 在浏览页任意焦点位置都可打开连接管理。
- 播放页默认焦点停在进度条，进度条按上可聚焦封面图；封面图获得焦点时显示直角高亮边框，OK 进入图片全屏预览。
- 收藏页进入空播放列表时，焦点回落到右上角返回按钮，避免遥控器落在不可操作空态文本上。
- 设置页左侧分类区分“当前已进入分类”和“遥控器焦点”：当前分类使用左侧蓝色竖线 + 低亮底色，焦点项使用亮蓝填充 + 明亮描边。
- 快速定位模式中，上下整屏跳，左右按 10% 分段跳，确认接受落点，返回取消。

### 缓存与恢复行为

- 浏览页按连接与目录路径保存目录焦点锚点；再次进入同一路径时优先恢复上次焦点，条目失效时回落到最近可用位置。
- 清理缓存会清空歌词缓存、封面缓存和浏览焦点锚点。
- 设置页“备份恢复”处理应用主 SQLite 数据库 `tsm-player.db`。该主库承载收藏/歌单、SMB 连接、浏览锚点、UI 设置、播放恢复状态、睡眠设置和 WebDAV 配置；歌词/封面磁盘缓存仍属于缓存，不进入备份范围。
- 老版本 `favorites.db`、`smb_config` DataStore 和各类 SharedPreferences 会在新表为空时渐进迁入 `tsm-player.db`；迁移后新读写以 `tsm-player.db` 为权威源。
- 本地导出会将当前 `tsm-player.db` 全量复制到应用备份目录；本地导入会先校验备份文件包含主库核心表结构，再覆盖当前 `tsm-player.db`。导入前必须弹出 confirm，文案说明会覆盖当前配置、播放状态与收藏数据，但不删除 SMB 原文件。
- WebDAV 上传会先生成本地全量主库备份，再以固定文件名 `tsm-player.db` 上传到配置的远端目录；WebDAV 下载会拉取同名文件到临时位置，校验通过后覆盖当前主库。上传/下载均由用户在设置页手动触发，不做自动同步、后台轮询或启动时自动恢复。
- 播放页支持继续上次播放；黄色按钮在无活跃播放且存在上次播放记录时显示“继续上次播放”。
- 播放页歌曲名、艺术家、专辑优先显示已解析的音频标签；播放/暂停、上一首/下一首、播放模式、睡眠定时返回、列表页“回到当前播放”等状态刷新不得把同一曲目的已解析标签回退为文件名、目录名等 fallback 信息。
- 列表页“继续上次播放”会恢复上次队列、曲目位置和播放进度，并立即启动播放，使播放服务在进入睡眠定时等二级页面后仍保持有效队列。
- 睡眠定时依赖后台播放服务存活；到点后保存播放快照、停止播放、退出应用，并通过设备管理权限请求电视息屏。未授权时仍会停播并退出应用。应用完全退出后重新冷启动时清除未完成的睡眠倒计时，但保留最近一次自定义时长。
- 歌词与封面支持磁盘缓存；播放页和全屏歌词页复用同一缓存键策略。
- 应用启动进入主界面时自动检查一次 GitHub 最新 Release；设置页“关于”分类下提供手动检查更新入口。

### 公共 TV Modal

- 全局弹层统一使用居中面板 + 深色遮罩（`ui_bg_overlay`），不再使用 Android 原生 `AlertDialog` / `ProgressDialog`。
- modal 统一分为标题区（section 标签 + 标题）、内容区（可滚动）、操作区（固定）；标题区和操作区固定，内容区可滚动。
- 五类 modal 复用同一套视觉 token 和焦点规则：
  - **Action Modal**：操作选择面板（如连接管理），内容区为命令按钮列表。
  - **List Modal**：列表选择面板（如切换连接），内容区为可选中行列表。
  - **Form Modal**：表单输入面板（如新建播放列表、SMB 配置），内容区为字段 + 行内错误，操作区为确认/次要按钮。
  - **Confirm Modal**：确认操作面板（如移除失效歌曲），消息区显示说明文案，操作区为确认/取消。
  - **Progress Modal**：进度展示面板（如下载更新），内容区为文件名 + 实时网速 + 下载量 + 蓝色确定进度条 + 说明文案。
- Back 关闭并恢复打开前焦点；遮罩层不可聚焦。
- 操作按钮使用 `isPrimary`（蓝色）/ `isDanger`（红色）/ 默认（深色）三档视觉。
- 表单字段支持文本输入（`TEXT`）和复选框（`CHECKBOX`）两种类型。
- 表单校验统一收口到 `TsmModalFormValidators`，返回 `null` 表示通过。
- `List Modal` 的相邻列表行之间保持 `ui_space_sm` 级别垂直间距，避免 TV 远距阅读时行块粘连。
- `Action Modal` 的操作项点击后默认关闭当前弹窗，避免下一层弹窗或页面状态更新后仍被旧操作面板遮挡。`List Modal` 行是否点击后关闭由调用场景决定：切换 SMB 连接应关闭，播放列表加入歌曲这类需要即时刷新列表状态的场景保持打开。
- 播放页长按收藏打开的“选择播放列表”弹窗在“加入现有播放列表”或“新建播放列表并加入”后，当前弹窗必须立即刷新列表状态，无需退出重进。

## Do's and Don'ts

- Do: UI 修改前先读取 `DESIGN.md`。
- Do: XML 通用样式使用 `@color/ui_*`、`@dimen/ui_*`、`@style/Tsm*` 和可复用 `@drawable/bg_*`。
- Do: 设置页分类保持 selected 和 focused 的视觉语义分离。
- Do: 修改页面级 UI 时同步更新 ASCII 草图和公共 token。
- Don't: 在 XML 中新增通用硬编码颜色、圆角、描边或字号。
- Don't: 调整 selector 时破坏 `state_focused + state_selected` 的优先级。
- Don't: 为 TV 工作流添加会抢走遥控器焦点的装饰性元素。
- Don't: 把图标、launcher 或一次性插画资产强行纳入通用 UI token。
