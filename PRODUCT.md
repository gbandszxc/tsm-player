# Product

## Register

product

## Users

Android TV 用户，坐在客厅沙发、距屏幕约 2–3 米，主要靠遥控器（D-Pad 上下左右 + OK + Back + Menu）操作，环境光偏暗（客厅、夜晚）。他们的 NAS/SMB 共享里存放个人音乐收藏（FLAC/MP3 等），希望在电视上低干扰地浏览目录、播放整库或单曲、收藏整理歌单、回看播放历史、设定睡眠定时，并查看同步歌词与封面。次要场景：debug 包可侧载到手机/平板做触控调试。共同的隐性约束是“远距离、暗光、单手遥控器”——任何设计都必须在这一物理场景下成立。

## Product Purpose

面向 Android TV（arm64-v8a + armeabi-v7a，Leanback）的 SMB 本地音乐播放器。它把家庭 NAS 上的音乐库搬到电视前，用遥控器完成“浏览 → 播放 → 收藏 → 历史 → 睡眠 → 歌词/封面”的完整链路，并能在后台保持播放、可中断可恢复。成功的标准：用户坐在沙发上能在 2–3 米外瞬间判断当前焦点、当前页面、播放状态和可执行操作；遥控器焦点始终稳定、可预测，不被装饰性元素或不可操作内容抢走；播放与浏览流程保持低干扰，设置、缓存、继续播放、定位都围绕“少打断、可恢复”设计。

## Brand Personality

克制为主，留一点温度。冷静的深蓝工程基底 + 蓝/绿/黄语义色 + MiSans 字体构成日常工具感；在封面、空状态、播放反馈等关键情感节点留出轻盈的插画与动效空间，呼应 launcher 上的二次元看板娘，但不把角色装饰铺进工具界面。三个词：克制、可靠、有温度。语气直接、不卖弄；像一个安静的家用音响，而不是喧闹的流媒体推荐墙。

## Anti-references

明确不想呈现的方向，作为后续改动的硬约束：

- **过浅/过白的现代 SaaS 风**：灰白卡片、大量留白、弱描边——在客厅暗光、远距离阅读时吃力，且与深色电视环境割裂。
- **过度装饰 / 赛博炫光**：霓虹紫粉、玻璃拟态、渐变发光、流动光效——廉价，且会抢走遥控器焦点。
- （隐含）**默认 Leanback 紫色模板感**与**流媒体音乐 App 的封面推荐墙**：缺乏品牌辨识度，且不适合逐项遥控操作。

## Design Principles

1. **焦点优先**。每个屏幕的首要任务是让遥控器用户在 2–3 米外瞬间识别当前焦点、页面与可执行操作。任何装饰、动画或不可操作内容都不得抢焦；焦点指示必须清晰且与“当前已进入状态”视觉分离。
2. **少打断、可恢复**。播放与浏览流程围绕“可中断、可恢复”设计：继续上次播放、定位当前曲目、睡眠定时、缓存清理都尽量不阻断主任务，状态可在页面重建或应用回到后台后恢复。
3. **克制有度**。工具界面保持冷静工程感，仅在关键情感节点（封面、空状态、播放反馈）释放一点温度，与 launcher 看板娘呼应，但不喧宾夺主。
4. **远距可读**。字号、对比度、触达尺寸都为客厅暗光、远距阅读服务；不为“精致留白”牺牲可读性，也不为装饰牺牲信息密度。
5. **表达一致**。同一语义的操作跨页面保持同一视觉词汇——黄色=恢复/定位/历史、绿色=播放/继续、红色=危险/不可逆、蓝色=焦点与主操作——差异只出现在必要处。

## Accessibility & Inclusion

以 **TV 遥控器可用性**为核心目标，不追求系统级全量无障碍：

- D-Pad 焦点链稳定、可预测，焦点指示清晰，焦点移动可自动滚动到可见区。
- 远距离暗光对比度按现有 `@color/ui_*` token 达标，正文与背景足够分明。
- 触控作为 debug 侧载到手机/平板时的兼容能力，不作为主要输入；不强求触屏目标尺寸的严格统一。
- 不在本阶段投入 TalkBack `contentDescription` 全量、动态字号缩放等系统级无障碍；后续如需可单独推进。
- 动效本就克制，`prefers-reduced-motion` 类需求在 TV 端影响面很小，保持当前节奏即可。
